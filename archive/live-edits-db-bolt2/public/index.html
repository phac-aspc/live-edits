Welcome to the Interactive Editor

This is a demonstration of a collaborative WYSIWYG editing system. Any section marked with the editable class can be modified by multiple users, with each change tracked and attributed.

The system uses Supabase for data storage, making it scalable and reliable. All edits are saved with timestamps and user initials, creating a complete audit trail of changes.

Technical Implementation

Architecture

The system consists of three main components:

  • Client Library: A JavaScript library that can be imported into any HTML page
  • API Backend: Supabase Edge Functions for handling data operations
  • Database: PostgreSQL tables with Row Level Security for storing edits and comments

How It Works

When you save changes, the entire HTML content is stored in the database along with your initials and a timestamp. The next time someone loads the page, they see the most recent version. Comments are stored with their exact coordinates, allowing them to be displayed as markers on the page.
